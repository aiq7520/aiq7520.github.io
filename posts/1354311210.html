登录修改密码

密码其实在 mysql/log/mysql_error.log中自动生成了。但是我发现登录不上去，现在让我们自己手动修改密码吧
在 mysql/etc/my.cnf 中添加 skip-grant-tables，并重启服务，并按下列步骤操作

  1. mysql；

  2. use mysql;

  3. 首先查看当前root用户相关信息，在mysql数据库的user表中；

    select host, user, authentication_string, plugin from user;

    表格中有以下信息：

    host: 允许用户登录的 ip ‘位置’ % 表示可以远程；

    user: 当前数据库的用户名；

    authentication_string: 用户密码（在mysql 5.7.9以后废弃了password字段和password()函数）；

    plugin： 密码加密方式；

  4. 将默认的 root 密码置空,后退出mysql客户端

    update user set authentication_string=’’ where user=’root’;

  5. 把配置文件 skip-grant-tables 注释掉或者删掉，重启服务。

  6. 重新登录到 MySQL 上

    mysql -u root -p

提示输入密码时直接敲回车，因为我们刚才已经将密码置为空了。

  1. 使用 ALTER 修改 root 用户密码

    ALTER user ‘root’@’localhost’ IDENTIFIED BY ‘aywoxEZX7d’;

远程连接

关于这个问题可以查看我的博客

  1. 修改user的host，修改成‘%’

    update user set host=’%’ where user =’root’;

    更新域属性，’%’表示允许外部访问

    FLUSH PRIVILEGES;

    再执行授权语句：

    GRANT ALL PRIVILEGES ON . TO ‘root’@’%’WITH GRANT OPTION;

    FLUSH PRIVILEGES; 命令本质上的作用是：
    将当前user和privilige表中的用户信息/权限设置从mysql库(MySQL数据库的内置库)中提取到内存里。
    MySQL用户数据和权限有修改后，希望在”不重启MySQL服务”的情况下直接生效，那么就需要执行这个命令。

    通常是在修改ROOT帐号的设置后，怕重启后无法再登录进来，那么直接flush之后就可以看权限设置是否生效。
    而不必冒太大风险。

  2. 检查linux是否开放3306端口号

    firewall-cmd –list-ports

  3. 对外开放3306端口号

    firewall-cmd –zone=public –add-port=3306/tcp –permanent

  4. 重启防火墙

    systemctl restart firewalld
